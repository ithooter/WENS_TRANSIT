# Транзитные документы — Веб-приложение. Роадмап

Цель: веб-приложение, в которое любой пользователь заходит, загружает Excel-«Исходник»,
выбирает нужные выходные документы (отдельные листы шаблона, весь шаблон, транзитный
документ, txt HSSC codes, CMR) с указанием Отправителя и Получателя, и скачивает результат.

**Стек:** Flask (Python) · SQLite · серверные HTML-шаблоны (Jinja2) + лёгкий JS.
**Запуск:** локально (`uvicorn`/`flask run`), деплой в облако — позже.
**Форматы Исходника:** `.xlsx`, `.xls`, `.xlsb` (конвертация `.xls/.xlsb` через LibreOffice).

Движки документов взяты из двух Python-проектов:
- **Excel Customs (WENS):** `parties.py`, `company_profiles.py`, `generate_customs_txt.py`,
  `hssc_txt_v3.py`, `fill_optiauto.py`, `fill_nova.py`.
- **Document packing for cars:** `generate_si_documents.py` (+ `classifier.py`).

## ТЕКУЩИЙ ФОКУС (по твоему запросу)

Сначала доводим до ума **три основных документа**, остальное — потом:

1. **INV (RU)** — инвойс с русскими наименованиями.
   - Сейчас: русское описание берётся из колонки источника, если она есть.
   - Будущее: подтягивать переводы через API (Спринт 3). Правило — не угадывать:
     нет перевода → ячейка пустая.
2. **PL** — упаковочный лист (Packing List).
3. **Invoice Transit** — транзитный инвойс (уже работает, проверен).

Когда эти три стабильны — добавляем остальное (TXT HSSC, CMR, отдельные листы) и тестируем.
