# Транзитные документы — веб-приложение

Веб-приложение для создания транзитных таможенных документов из Excel-«Исходника».
Пользователь логинится, загружает файл, выбирает нужные документы (отдельные листы
шаблона, весь шаблон, транзитный документ, txt HSSC codes, CMR), указывает
Отправителя и Получателя — и скачивает результат.

Стек: **Flask + SQLite + Jinja2**. Движки документов — Python (pandas/openpyxl),
перенесены из двух проектов: *Excel Customs (WENS)* и *Document packing for cars*.

> Это **Спринт 0 — фундамент**. План развития: см. [ROADMAP.md](ROADMAP.md).

## Что уже работает

- Регистрация / вход / выход (без верификации e-mail).
- Загрузка `.xlsx / .xls / .xlsb`.
- Экран выбора 5 типов документов + поля Отправитель/Получатель
  с автоподстановкой из реестра `parties.py`.
- Генерация и скачивание:
  - **Весь шаблон** — отдаёт загруженный файл (нормализованный в .xlsx).
  - **Отдельные листы** — каждый лист отдельным .xlsx.
  - **Транзитный документ** — `INVOICE_TRANSIT_*.xlsx` через движок SI.
  - **TXT HSSC** — TXT для таможни Дубая (нужен лист `HS CODE SUM`/`HSSC`/`INV`).
  - **CMR** — таблица для накладной (.xlsx; формат .doc — в Спринте 4).

## Два пробела (нужны от тебя — см. ROADMAP, Спринт 1–2)

1. **`app/engines/classifier.py`** — сейчас заглушка. Замени своим реальным
   классификатором ТНВЭД (ключевые слова + HS-коды). Без него транзитный
   документ группирует по «HS-код + описание», а не по товарным группам.
2. **Актуальный `fill_optiauto.py`** — версия с параметрами `inv_enrichment`
   и `pl_dedup_field` (их ждёт `fill_nova.py`). Нужна для режима «заполнить
   шаблон» в Спринте 2.

## Заметки по безопасности (перед публичным деплоем — Спринт 5)

- Задать `SECRET_KEY` через переменную окружения.
- Папка `instance/` не коммитится (БД и файлы пользователей).
- Перед деплоем: HTTPS, лимит размера файла, очистка старых файлов, rate-limit.
